Когда использовать DuckDB


  • Обработка и хранение наборов табличных данных, например из файлов CSV или Parquet
  • Интерактивный анализ данных, например объединение и агрегирование нескольких больших таблиц
  • Одновременные большие изменения в нескольких больших таблицах, например добавление строк, добавление/удаление/обновление столбцов
  • Передача большого набора результатов клиенту

Когда не использовать DuckDB


  • При больших объемах транзакций (например, отслеживание заказов в интернет-магазине)
  • Крупные клиент-серверные установки для централизованного хранения корпоративных данных
  • Запись в одну базу данных из нескольких параллельных процессов
  • Несколько одновременных процессов чтения из одной доступной для записи базы данных
